Bootstrap Utility Classes Complete Reference

Utility classes are single-purpose classes that do one thing well. They follow a consistent naming pattern.

Utility Class Structure:

Most utility classes follow this pattern: {property}{sides}-{size}

  • property: What you're setting (m for margin, p for padding, etc.)
  • sides: Which sides (t for top, b for bottom, x for left+right, etc.)
  • size: How much (0-5, auto, or responsive breakpoint)

Spacing Utilities (Margin & Padding)

Class What it means Example
m-* Margin on ALL sides m-3 = 1rem margin all around
mt-* Margin TOP only mt-2 = 0.5rem top margin
mb-* Margin BOTTOM only mb-4 = 1.5rem bottom margin
ms-* (start) Margin LEFT (in LTR languages) ms-3 = 1rem left margin
me-* (end) Margin RIGHT (in LTR languages) me-2 = 0.5rem right margin
mx-* Margin LEFT & RIGHT (horizontal) mx-4 = 1.5rem left & right margin
my-* Margin TOP & BOTTOM (vertical) my-3 = 1rem top & bottom margin
p-* Padding on ALL sides p-3 = 1rem padding all around
pt-* Padding TOP only pt-2 = 0.5rem top padding
pb-* Padding BOTTOM only pb-4 = 1.5rem bottom padding
ps-* Padding LEFT (in LTR languages) ps-3 = 1rem left padding
pe-* Padding RIGHT (in LTR languages) pe-2 = 0.5rem right padding
px-* Padding LEFT & RIGHT (horizontal) px-4 = 1.5rem left & right padding
py-* Padding TOP & BOTTOM (vertical) py-3 = 1rem top & bottom padding
Size scale: 0=0rem, 1=0.25rem, 2=0.5rem, 3=1rem, 4=1.5rem, 5=3rem
Auto: mx-auto centers horizontally
Responsive: mt-md-3 applies only on medium screens and up

Grid & Display Utilities

col-sm-* Column on SM screens & up col-sm-4 = 4 columns on small+
col-md-* Column on MD screens & up col-md-3 = 3 columns on medium+
d-* Display property d-flex, d-none, d-block
d-sm-* Display on SM screens & up d-none d-md-block = hide on mobile, show on medium+

Grid breakpoints: sm (≥576px), md (≥768px), lg (≥992px), xl (≥1200px), xxl (≥1400px)

Customizing Bootstrap

Learn how to override Bootstrap's default styles to match your design.

Method 1: CSS Overrides

Create your own CSS file that loads AFTER Bootstrap and override styles:

/* custom.css - Load this AFTER bootstrap.css */

/* Change primary color */
.btn-primary {
    background-color: #6f42c1;
    border-color: #6f42c1;
}

.btn-primary:hover {
    background-color: #5936a0;
    border-color: #5936a0;
}

/* Change navbar */
.navbar {
    border-bottom: 3px solid #6f42c1;
}

/* Increase border radius */
.rounded {
    border-radius: 10px !important;
}

/* Custom shadow */
.shadow {
    box-shadow: 0 5px 15px rgba(111, 66, 193, 0.2) !important;
}

Method 2: CSS Variables (Recommended)

Bootstrap 5 uses CSS custom properties that you can override:

/* Override in your CSS file */
:root {
    /* Color system */
    --bs-primary: #6f42c1;
    --bs-primary-rgb: 111, 66, 193;
    --bs-secondary: #ea39b8;
    --bs-success: #198754;
    
    /* Typography */
    --bs-font-sans-serif: 'Inter', system-ui, -apple-system, sans-serif;
    --bs-body-font-size: 1.1rem;
    
    /* Borders */
    --bs-border-radius: 0.5rem;
    --bs-border-radius-sm: 0.25rem;
    
    /* Spacing */
    --bs-spacer: 1.2rem;
    
    /* Shadows */
    --bs-box-shadow: 0 5px 15px rgba(0, 0, 0, 0.1);
}

Method 3: Sass Customization (Most Powerful)

For complete control, customize Bootstrap's Sass variables before compiling:

// custom.scss

// 1. Override default variables
$primary: #6f42c1;
$secondary: #ea39b8;
$enable-rounded: true;
$border-radius: .5rem;
$font-family-base: 'Inter', system-ui, -apple-system, sans-serif;

// 2. Import Bootstrap
@import "~bootstrap/scss/bootstrap";

// 3. Add custom styles
.custom-component {
    background-color: tint-color($primary, 90%);
    border-left: 4px solid $primary;
}

// Create your own utility classes
.bg-gradient-primary {
    background: linear-gradient(135deg, $primary 0%, darken($primary, 15%) 100%);
}
